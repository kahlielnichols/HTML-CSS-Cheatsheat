Git Cheat Sheet

What is Git?

Git is the most widely used modern version control system in the world today.

Performances

Git take advantage of deep knowledge about common attributes of real source code file trees, how they are usually modified over time and what the access patterns are.

Being distributed enables significant performance benefits as well.

Security

Git has been designed with the integrity of managed source code as a top priority.

Some other version control systems have no protections against secret alteration at a later date

Flexibility

One of Git's key design objectives is flexibility. Git is flexible in several respects:

In support for various kinds of nonlinear development workflows, in its efficiency in both small and large projects and in its compatibility with many existing systems and protocols.

Version control with Git

Git is the best choice for most software teams today.

Git is good

Git has the functionality, performance, security and flexibility that most teams and individual developers need.

Repository

The overall package or project. There is typically a remote of origin respository, and a local copy of it at certain time on your local machine

Branch

An extension of a repository. All repositories, or repos,start with a 'master' branch. From there, new branches can be created and used for new features. The idea is to keep the master branch clean and working for the final " done" project.

Commit

a commit is a change to the repository. For every commit made, there is a recod of the state of the repository at that given point in time.
